PHP中如何返回上一级目录
2014-04-04 ⁄ PHP技术 

正式入职的第二天，搞着恶习的PHP,不，不应该加恶心两字，语言本身没有什么好坏，语言的创建只是为了更好服务于人，看了看项目里的代码，有这么一句

$fpdir = realpath(dirname(__FILE__).'/../..').'/';

初看之下，从函数名中就可以看出该语句的实际作用，可/../..是啥意思呀？我们在shell中返回上一级目录的命令如下所示：

cd ../../

这个/../..是什么意思呀，PHP里搞的是什么飞机呀?看了看realpath的帮助文档 http://www.php.net/manual/zh/function.realpath.php 里面是这么写的

realpath() 扩展所有的符号连接并且处理输入的 path 中的 '/./', '/../' 以及多余的 '/' 并返回规范化后的绝对路径名。返回的路径中没有符号连接，'/./' 或 '/../' 成分

读起来相当的绕口，是吧，读了半天不知道其想表示什么意思，即然不知道其表示的是什么意思，那么就看看PHP源码里是怎么实现的吧

在看PHP源码之前，我猜realpath的实现是将参数string以/进行分割，分割后的结果放到一个栈里面，然后再出栈，在出栈的过程中如果遇到..则将..下面的目录也弹出来,表示返回上一级目录，如果遇到.则只弹出.，表示仍在当前目录下，如果学习过Java的，此处很容易想到java里的一个函数split，这个函数就是通过指定一个特殊的字符，以这个特殊的字符将字符串分割，分割后的字符串保存在一个数组里面

哎，找了很久的PHP里面该代码是怎么实现的，也没有发现在哪，只发现了一行帮助不大的PHP内核里面C语言代码：

TSRM_API char *realpath(char *orig_path, char *buffer)
{
    int ret = GetFullPathName(orig_path, _MAX_PATH, buffer, NULL);
    if(!ret || ret > _MAX_PATH) {
        return NULL;
    }
    return buffer;
}

因此以下代码的意思就是

$fpdir = realpath(dirname(__FILE__).'/../..').'/';

从当前文件所在目录路径向上返回两级，然后在目录路径后面加上/

如__FILE__表示的是

D:\workspace\Hello\hello.php

则dirname函数作用后D:\workspace\Hello

再由realpath(dirname(FILE).'/../..')作用后返回的结果是

D:\

因此PHP中获取目录上一级的代码可以用realpath或者dirname实现

如下两段代码在意义上是等价的

$fpdir = realpath(dirname(__FILE__).'/../..').'/';
echo  $fpdir;


$dir=dirname(dirname(dirname(__FILE__))).'/';
echo $dir;
